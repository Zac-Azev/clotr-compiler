CLOTR ideation - SaM compiler in Lord of the Rings style

Ideas are ranked in importance order from top to bottom. If, to fulfill a higher rule, we must throw away a lesser one, so be it.

- MUST follow all of Saad's requisites, but implement the BARE MINIMUM HE ASKS (at most, some extra functions and NOTHING MORE to make it EASY to implement the actually FUN part)
- Must look like a narrative writing tale with dnd vibes and races as name types
(such as a Hobbit for an int)
- gotta have "free writing zones": spaces where, as the command is already written, parser can ignore text until next relevant information appear.
(example: a free writing zone between the full if declaration and the ":" that closes it. Any text between the second part of the comparison and the ":" is ignored
"if Frodo equals Sam, oops! such a thing would never happen, silly writer:", ", oops! such a thing would never happen, silly writer" would be ignored)
- free writing zones declarable as a #editor-note, such as #defines in C. The syntax for a definition should be as follows: #editor-note [line where there's an ignore] [free writing zone starting character] [free writing zone last character]
- if such a rule initiates, the line breaking would be important for text creation.
